security

Your safety and privacy are our priorities.

The reVisit source code is opened and you can verify that there are no "bookmarks". The server does not store passwords in clear text and does not have the ability to decrypt them.

There is the possibility of both uncontrolled access, for example, remote servers, and manual confirmation of each connection.

In order to prevent unauthorized changes to the program code, reVisit is signed by the Code Signing Developer Certificate. What guarantees you the authenticity of the received program.

The remote access manager reVisit supports the operation of any VNC client, if you have preferences, you can use any version, including commercial development.
